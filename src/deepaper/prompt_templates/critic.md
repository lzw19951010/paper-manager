## 审核清单

对每个章节检查以下维度：

### 1. 完整性
模板要求的每个子项是否存在且有实质内容（不是敷衍一句话）：
- 核心速览: TL;DR(含数字) + 一图流(类比) + 核心机制一句话([动作]+[对象]+[方式]+[效果])
- 动机与第一性原理: 痛点(引用baseline数字) + 因果链(Because→Therefore) + 物理/直觉解释
- 方法详解: 直觉版(引用Figure) + 精确版(流程图+公式+数值推演+伪代码) + 设计决策 + 易混淆点(❌/✅)
- 实验与归因: 对比表格 + 归因排序 + 可信度检查
- 专家批判: 隐性成本(含具体数字) + 工程落地建议 + 关联思考
- 机制迁移分析: 解耦表格(4列) + 迁移处方(4要素) + 家族图谱(前身≥3)
- 背景知识补充: 外部技术说明

### 2. 风格
- "费曼技巧"是否体现：用直觉、类比、大白话解释机制，而非堆砌术语
- 数值推演是否用了具体数字走一遍（而非只列公式）
- 一图流是否真的给了一个直观类比（而非重复 TL;DR）

### 3. 准确性
- 数字是否看起来合理（不是编造的整数，如"提升了约50%"）
- 公式符号是否有完整的物理含义解释
- Baseline 对比是否公平（不只挑弱的比）

### 4. 标注【必做】的项
- 数值推演（方法详解/精确版）
- 伪代码（方法详解/精确版）

## 输出格式

只审核，不修改。输出 JSON：

```json
{
  "sections": {
    "核心速览": {
      "score": 8,
      "present": ["TL;DR含数字", "一图流类比"],
      "missing": ["核心机制一句话缺少[动作]+[对象]格式"],
      "comment": "TL;DR 数字充实，但一图流和TL;DR内容重复"
    },
    "动机与第一性原理": {
      "score": 7,
      "present": ["痛点引用了baseline数字", "因果链存在"],
      "missing": ["物理/直觉解释过于简略"],
      "comment": "因果链清晰但物理解释只有一句话"
    }
  },
  "style_check": {
    "feynman_technique": true,
    "no_jargon_piling": true,
    "concrete_numbers": true,
    "comment": "整体风格符合费曼技巧要求"
  },
  "overall_score": 7,
  "critical_issues": [
    "方法详解缺少数值推演（模板标注【必做】）",
    "专家批判的隐性成本没有具体数字"
  ]
}
```

评分标准：
- 10: 完美遵循模板所有要求
- 7-9: 主要内容齐全，有小缺陷
- 4-6: 部分章节缺失或敷衍
- 1-3: 严重偏离模板要求
- overall_score < 6 会触发 Fixer 修复
